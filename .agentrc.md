# Agent Onboarding Instructions

Welcome to the **VimDocs** project! This codebase implements Vim keybindings for Google Docs via a Chrome Extension.

Before modifying code, read this document fully to avoid falling into known traps or breaking the architecture.

## Context & Limitations
1.  **Google Docs Environment**: Google Docs uses a highly custom rendering engine (canvas-based). Standard DOM text manipulation does not work. We rely on simulating key events (`KeyboardEvent`) and dispatching them to the main `iframe`.
2.  **Clipboard Access**: We **cannot** programmatically access the OS clipboard from within the Google Docs iframe due to strict browser security policies (`document.execCommand('copy')` fails). 
    *   **The Fix**: For yanking (`y`) and pasting (`p`), we rely on native pass-through. Pressing `y` must immediately show a "USE CTRL+C" message without performing any text selection or waiting for motions. **Never attempt to implement bootstrapped selection-based yanking.**
3.  **Search Functionality**: Similar to clipboard access, we **cannot** reliably implement a custom Vim Search mode that interacts with the native Docs find UI or the Canvas-rendered text.
    *   **The Fix**: Delegate search to native Docs (`Ctrl+F`, `Ctrl+G`, `Ctrl+H`). For Vim-style search keys (`/`, `?`, `n`, `N`), we show a guidance toast and do nothing else. **Always allow search-related Ctrl shortcuts to pass through un-intercepted.**
4.  **Real-Time Settings**: The options page uses an auto-save pattern with immediate persistence to `chrome.storage.sync`. 
    *   **The System**: `options.js` triggers `saveOptions()` on every interaction. `content.js` reacts via `chrome.storage.onChanged`.
    *   **Live Preview**: For Status Line size, a separate `VIM_DOCS_SIZE_PREVIEW` runtime message is sent for instantaneous visual feedback without disk I/O. **Do not re-introduce a manual Save button.**

## Architecture Overview
*   **`src/content.js`**: The main brain. It injects the mode indicator UI, listens for keyboard events, manages the state machine (Modes: Normal, Insert, Visual), and interprets Vim commands. It also handles dynamic keybindings loaded from Chrome Storage.
*   **`src/emulator.js`**: Reusable utility functions for dispatching specific key events (e.g., `moveLeft`, `deleteChar`, `selectLine`).
*   **`options.html` / `src/options.css` / `options.js`**: The settings UI for customizing keybindings and enabling/disabling the extension.
*   **Vite**: The project is built using Vite and `crxjs`. After making changes to the UI or Content scripts, you must run `npm run build`.

## Design System
*   We use a **Neon-Dark Aesthetic** (see `DESIGN_SYSTEM.md`).
*   **Colors**: Backgrounds are `#050505` or glassmorphic (`rgba(31,31,31,0.8)`). Accents and interactive states must use Neon Green (`#CCFF00`).
*   Do not introduce generic "Bootstrap" style colors or Tailwind defaults. Stick strictly to the defined CSS variables in `src/options.css`.
